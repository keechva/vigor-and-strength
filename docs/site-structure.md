# Структура сайта «Бодрость и Сила»

Информационная архитектура и контент-план фазы 1. Без вёрстки и кода — только структура страниц, описание блоков и связи между ними.

---

## Навигация (основное меню)

```
Главная · Баня · Релаксология · Тренировки · Мастера · Локации
```

Мастера доступны через меню и через карточки внутри разделов. QR-коды на визитках ведут напрямую на `/masters/[slug]` — страница мастера работает как самостоятельная точка входа.

---

## Карта сайта

```
/                     Главная (лендинг)
/banya                Баня
/relax                Релаксология
/trainings             Тренировки (кунгфу, медитации, работа с дыханием)
/masters              Все мастера
/masters/[slug]       Страница мастера
/locations            Локации (хаб с карточками)
/locations/[slug]     Страница локации
/blog                 Статьи (фаза 1.5, опционально)
/blog/[slug]          Статья
```

## Страницы

### / — Главная

Лендинг. Знакомит с проектом и направляет дальше. Формы записи нет — только атмосфера и ориентиры.

#### Блок 1: Hero
- Полноэкранный или крупный визуальный блок.
- Фото или видео, передающее атмосферу: тело, пар, движение.
- Название проекта: **Бодрость и Сила** (крупно).
- Под названием — три направления списком: **Баня · Релаксология · Тренировки**.
- Без hero-фразы: название и список направлений говорят сами.
- Без кнопки записи — пространство для погружения, не продажа.

#### Блок 2: Три направления
- Три карточки или секции: **Баня**, **Релаксология**, **Тренировки**.
- Каждая — краткое описание (1–2 предложения) + ссылка на раздел.
- Без перечисления услуг, без цен. Только ощущение.
- Можно сочетать — между блоками деликатная перекрёстная связь: «хорошо работает вместе».

#### Блок 3: Превью мастеров
- Лента из 4 курируемых карточек — даёт лица проекту.
- Карточка: фото, имя, теги направлений, короткая фраза.
- Каждая карточка → `/masters/[slug]`.
- Ссылка-якорь «Все мастера» → `/masters`.
- Состав превью выбирается вручную — не все мастера попадают на главную.

#### Блок 4: О нас
- Кто мы: команда, сообщество, подход.
- Что даём — без маркетинговых клише, живым языком.
- Мастера упоминаются как люди, не как «специалисты».
- Не объясняем зачем нужна баня или кунгфу — говорим о себе.

#### Блок 5: Соцсети
- Ссылки на активные каналы/группы.
- Короткий призыв: подписывайся, следи за нами.
- Можно вставить последний пост или фото как превью.

#### Блок 6: Поддержка
- Тон: мы в начале пути, нам важен каждый.
- Призыв: расскажи друзьям, поделись, запишись сам — это и есть поддержка.
- Без пафоса и давления. Короткий, человечный текст.
- Можно добавить: «Каждый подписчик, каждая запись — для нас это важно».

#### Блок 7: Футер
- Логотип, название.
- Навигация (дублирует меню).
- Ссылки на соцсети.
- Placeholder контактов (заменим перед публичным запуском).
- Юридический плейсхолдер — ИНН, адрес, форма собственности (реальные данные позже).

### /banya — Баня

Страница направления. Рассказывает что такое банное направление, что входит, сколько стоит, кто работает.

#### Блок 1: Описание направления
- Заголовок: **Баня**.
- 2–4 предложения о том, что это и как это происходит. Без объяснений зачем.
- Форматы: выезд к клиенту, партнёрские бани, собственная локация.
- Вместимость: 1–15 человек, возможна работа в 4 руки.

#### Блок 1а: Ориентир по цене
- Короткая фраза, снимает страх «нет цен — значит дорого».
- Формулировка: «Цена согласуется индивидуально — соберите состав в конфигураторе, и мы назовём ориентир». [уточнить финальную формулировку]
- Не прайс, не вилка — ссылка на конфигуратор как инструмент расчёта.

#### Блок 2: Базовое парение
- Название: **Базовое парение**.
- Описание: приход, работа с паром, веник.
- Это основа — на что всё остальное нанизывается.

#### Блок 3: Допники
- Список дополнительных опций с кратким описанием каждой:
  - Выбор веника (берёза, дуб, эвкалипт, хвоя и др.)
  - Скрабирование
  - Ароматерапия
  - Звукотерапия (поющие чаши)
  - Чай / травяной напиток
  - Работа с телом после пара (тактильное восстановление)
- Не прайс — описание ощущений и смысла каждого.

#### Блок 4: Программы
- Готовые комбинации для тех, кто не хочет собирать вручную:

| Программа | Состав |
|---|---|
| Классика | Парение + веник на выбор + чай |
| Восстановление | Парение + скрабирование + работа с телом после пара + чай |
| Сенсорная | Парение + ароматерапия + звукотерапия |
| В 4 руки | Парение + работа с телом двумя мастерами |
| Полный релакс | Все допники включены |
| Компания | До 15 человек, базовое + опции под группу |

#### Блок 5: Конфигуратор стоимости
- Цена зависит от: количества человек + набор допников.
- Интерактивный элемент: выбираешь количество людей и допники — видишь итоговую цену.
- Не жёсткий прайс-лист — гибкий расчёт.
- После конфигурации — кнопка «Оставить заявку» с передачей выбранного состава в форму.

#### Блок 6: Мастера этого направления
- Лента карточек пар-мастеров.
- Карточка: фото, имя, короткое описание, ссылка на `/masters/[slug]`.
- Не вложенная структура — мастера показываются горизонтально рядом с услугами.

#### Блок 7: Форма заявки
- Заголовок: «Записаться».
- Поля: имя, телефон, пожелания (опционально), выбранная программа (если пришёл из конфигуратора — подставляется автоматически).
- Заявка уходит в Telegram-бот, мы перезваниваем.
- Никаких внешних систем записи на этом этапе.

#### Перекрёстные ссылки
- «Хорошо сочетается с релаксологией» → /relax
- Карточки мастеров → /masters/[slug]
- Программы с упоминанием локаций → /locations

#### Блок 2: Кунгфу

### /masters/[slug] — Страница мастера

Самостоятельная точка входа. Работает через QR-код на визитке без необходимости знать остальной сайт.

#### Блок 1: Профиль
- Имя мастера (крупно).
- Фото.
- Теги направлений: Баня / Релаксология / Тренировки. На карточке могут уточняться конкретные практики (кунгфу, медитации, работа с дыханием), но направление-тег — «Тренировки».
- Описание: кто такой, подход, характер работы. Живым языком, от первого лица.

#### Блок 2: Услуги
- Заголовок: **Чем занимаюсь**.
- Список берётся из данных **конкретного мастера** (поле `services` в его файле данных) — не генерируется из общего пула направления.
- Это принципиально: если в направление добавляется новая услуга, страницы мастеров не меняются. Каждый мастер явно перечисляет своё.
- Без прайса — описания.

#### Блок 3: Форматы работы
- Выезд к клиенту / партнёрская локация / собственная локация.
- Ссылки на конкретные локации если применимо → /locations.

#### Блок 4: Форма заявки
- Заголовок: «Записаться к [имя]».
- Поля: имя, телефон, пожелания.
- Заявка идёт в Telegram-бот с пометкой мастера.

#### Перекрёстные ссылки
- Теги направлений кликабельны → /banya, /relax, /trainings.
- Локации кликабельны → /locations.

### /locations — Локации (хаб)

Хаб с карточками-превью всех локаций. Масштабируется без переделки: добавить локацию = добавить файл данных. Партнёры получают прямую ссылку на свою страницу.

#### Блок 1: Заголовок и фильтр
- Заголовок: **Локации**.
- Фильтр: **Все · Свои · Партнёрские · Выезд**.

#### Блок 2: Сетка карточек-превью
Карточка локации содержит: фото, название, тип (своя/партнёрская/выезд), краткое описание (1–2 предложения), теги доступных направлений, кнопка «Подробнее» → `/locations/[slug]`.

Текущие локации:
- **Самолётная 59** — своя, в ремонте. Баня до 4 чел, тренировки, релаксология.
- **Лукоморье** — партнёрская, усадьба за городом. До 15 чел, премиум. Баня.
- **Китуся** — партнёрская, в черте города. До 8 чел, бюджетные. Баня.
- **Оазис отдыха** — партнёрская, за городом. До 20 чел, домики с купелями. Баня.
- **Выезд к клиенту** — без адреса. Релаксология (основной), баня (при наличии у клиента).

#### Блок 3: «Нет подходящего?»
- Текст: «Работаем на выезд, обсудим твой вариант».
- Форма заявки.

#### Перекрёстные ссылки
- Карточка локации → `/locations/[slug]`.
- Теги направлений на карточке → `/banya`, `/relax`, `/trainings`.

### /locations/[slug] — Страница локации

Шаблонная страница. Аналог страницы мастера — самостоятельная точка входа, партнёры могут давать прямую ссылку на свою локацию.

#### Блок 1: Шапка локации
- Название (крупно).
- Фото или галерея.
- Тип: своя / партнёрская / выезд.
- Статус (если актуально): «в разработке», «открыто».
- Адрес и краткое описание места (атмосфера, уровень, особенности).

#### Блок 2: Вместимость и форматы
- Количество человек (мин–макс).
- Доступные форматы: группа / пара / индивидуально / 4 руки.

#### Блок 3: Доступные направления
- Теги направлений, которые здесь работают: [Баня] [Релаксология] [Тренировки].
- Каждый тег кликабелен → `/banya`, `/relax`, `/trainings`.

#### Блок 4: Мастера на этой локации
- Карточки мастеров, работающих на локации (фото, имя, направление).
- Каждая карточка → `/masters/[slug]`.

#### Блок 5: Форма заявки
- «Хочу сюда» / «Записаться на [название]».
- Поля: имя, телефон. Скрытый контекст: slug локации.

#### Перекрёстные ссылки
- Теги направлений → `/banya`, `/relax`, `/trainings`.
- Карточки мастеров → `/masters/[slug]`.
- Ссылка «Все локации» → `/locations`.

### /blog — Статьи (фаза 1.5)

Необязателен на старте. Структура заложена, включается без переделки остального сайта.

- Список статей/событий в обратном хронологическом порядке.
- Карточка: заголовок, дата, короткое превью, тег (баня / релаксология / кунгфу / событие).
- Статьи в MDX — редактируются как текстовые файлы, без CMS.
- Контент пишется как черновик, проходит ревью, только потом публикуется.

## Форма заявки

Единый механизм CTA. Присутствует на страницах: направлений (Баня, Релаксология), мастеров, локаций.

- **Поля:** Имя, Телефон, Пожелания (опционально).
- **Контекст:** передаётся скрытым полем — откуда пришла заявка (направление, мастер, локация, программа из конфигуратора).
- **Обработка:** заявка уходит в Telegram-бот, менеджер перезванивает и согласовывает время вручную.
- **Никаких внешних систем записи** на этапе MVP.

Тренировки — отдельный лист ожидания: имя + телефон/email, уведомление когда откроется запись.

## Карта страниц: статические и шаблонные

### Статические страницы (одна страница = один URL)

| URL | Название | Тип |
|---|---|---|
| `/` | Главная | Статическая |
| `/banya` | Баня | Статическая |
| `/relax` | Релаксология | Статическая |
| `/trainings` | Тренировки | Статическая |
| `/masters` | Все мастера | Статическая |
| `/locations` | Локации (хаб) | Статическая |
| `/blog` | Статьи (фаза 1.5) | Статическая |

### Шаблонные страницы (один шаблон, много инстансов)

| Шаблон | URL-паттерн | Источник данных |
|---|---|---|
| Страница мастера | `/masters/[slug]` | MDX-файл или JSON на мастера |
| Страница локации | `/locations/[slug]` | MDX-файл или JSON на локацию |
| Статья | `/blog/[slug]` | MDX-файл на статью |

**Страница мастера** — шаблон с фиксированной структурой блоков, контент различается. При добавлении нового мастера создаётся новый файл данных — никаких изменений в коде.

**Страница локации** — аналогично мастеру: один шаблон, файл данных на каждую локацию. Партнёры получают прямую ссылку. Масштабируется до любого числа локаций без переделки вёрстки.

## Конфигуратор бани

Живёт как блок внутри страницы `/banya` — не отдельная страница. Пошаговый выбор состава сеанса.

### Шаг 1: Количество человек
- Выбор: 1, 2–4, 5–8, 9–15 человек
- Влияет на базовую ориентировочную стоимость

### Шаг 2: Выбор базы
- Только базовое парение
- Базовое парение + веник (выбор веника: берёза / дуб / эвкалипт / хвоя)

### Шаг 3: Допники (мультивыбор)
- Скрабирование
- Ароматерапия
- Звукотерапия (поющие чаши)
- Чай / травяной напиток
- Работа с телом после пара

Альтернатива шагам 2–3: **выбрать готовую программу** (Классика / Восстановление / Сенсорная / Полный релакс / В 4 руки / Компания). Программа заполняет шаги автоматически — можно дальше скорректировать.

### Шаг 4: Итог
- Краткий список выбранного
- Ориентировочная стоимость: «от X рублей — точную сумму согласуем при звонке»
- Кнопка «Оставить заявку»

### Шаг 5: Заявка (финальный)
- Форма: имя + телефон
- Весь состав из конфигуратора передаётся автоматически
- Отправляется в Telegram-бот

**Важно:** конфигуратор не блокирует запись без прохождения шагов. На странице бани есть и обычная форма заявки для тех, кто не хочет настраивать.

## Форма заявки: механика

### Типы форм на сайте

**1. Обычная форма** (Баня, Релаксология, Мастер, Локации)
- Поля: Имя, Телефон
- Скрытый контекст: тип страницы + идентификатор (slug мастера / название направления)
- Результат в Telegram: «Заявка с [страницы]: Имя — Телефон»

**2. Конфигуратор-форма** (финальный шаг конфигуратора бани)
- Поля: Имя, Телефон
- Скрытый контекст: весь состав (количество человек, база, допники, программа, ориентировочная цена)
- Результат в Telegram: структурированное сообщение с составом

**3. Форма ожидания / лист ожидания** (Тренировки)
- Поля: Имя, Телефон или Email
- Результат в Telegram: «Ожидает записи на кунгфу: Имя — Контакт»

### Где появляется форма
- Внутри страницы (не попап, не отдельная страница) — блок в нижней части раздела
- На странице мастера — закреплённый блок в конце
- На странице локации — в разделе «Нет подходящего? Обсудим»
- После конфигуратора — встроена как финальный шаг

## Контент-план

### Принципы тона (все страницы)
- **Тихая уверенность того, кто делает давно.** Экспертность через интонацию, не через «20 лет опыта».
- Без обещаний и маркетинговых клише. Никаких «предлагаем широкий спектр услуг».
- Не объясняем зачем нужна баня или тренировки — человек сам знает.
- Живые ощущения, не характеристики. Не «продолжительность 60 минут», а «вечер, который вы себе устраиваете».
- Обращение — на «вы».
- Голос — описательный (что есть, что происходит). Исключения: блоки «О нас» и «Поддержка» на главной — допустимо «мы»; страница мастера — от первого лица «я».
- Слово «массаж» не используется нигде.

### Главная страница
- **Hero:** только название «Бодрость и Сила» крупно + список трёх направлений. Без hero-фразы.
- **Блок направлений:** 1–2 предложения на каждое. Ощущение, не перечень.
- **О нас:** 3–5 предложений. Кто мы, как появились, что важно. Не пресс-релиз.
- **Поддержка:** 2–3 предложения. Тепло, без давления.

### Страница направления
- **Шапка:** 2–4 предложения. Что это, как происходит, что чувствуешь.
- **Базовая услуга:** 3–5 предложений. Описание опыта.
- **Допники:** 1–3 предложения на каждый. Смысл, не инструкция.
- **Программы:** таблица + 1 предложение на программу.

### Страница мастера (шаблон — заполняет мастер или редактор)
- **Описание:** 3–6 предложений от первого лица. Кто такой, как работает, что важно в работе.
- **Запрещено:** «Сертифицированный специалист с опытом 5 лет» — безличный корпоративный стиль.
- **Обязательно:** что-то личное — откуда пришёл к этому, что ценит в работе.
- **Услуги:** краткий список, без прайса.

## Модель данных

> **Открытый вопрос (issue #5):** формат хранения данных (MDX с frontmatter / JSON / TypeScript-объект) ещё не утверждён. Структура полей ниже согласована; формат решается перед началом вёрстки страницы мастера.

### Мастер

| Поле | Тип | Описание |
|---|---|---|
| `name` | строка | Отображаемое имя |
| `slug` | строка | URL-идентификатор (`/masters/[slug]`) |
| `photo` | путь к файлу | Основное фото |
| `directions` | массив строк | Теги направлений: `banya`, `relax`, `trainings` |
| `role` | строка | Роль в команде: `lead` (ведущий мастер) / `master` (самостоятельный) / `student` (ученик). По контент-черновику: Дмитрий — `lead`, Александр — `master`, остальные — `student` |
| `featuredOnHome` | булево | Показывать ли карточку в превью мастеров на главной (4 курируемых) |
| `description` | текст | Описание от первого лица, 3–6 предложений |
| `services` | массив id | Личный набор услуг — id из общего пула направления. Задаётся явно, не наследуется автоматически |
| `formats` | массив строк | Форматы работы: `visit` (выезд), `partner` (партнёрская), `own` (своя локация) |
| `locations` | массив slug | Локации, на которых работает мастер |

**Важно про `services`:** каждый мастер явно перечисляет свои услуги. Добавление новой услуги в пул направления не затрагивает существующие страницы мастеров.

### Локация

| Поле | Тип | Описание |
|---|---|---|
| `name` | строка | Название |
| `slug` | строка | URL-идентификатор (`/locations/[slug]`) |
| `type` | строка | `own` / `partner` / `visit` |
| `status` | строка | `open` / `in-progress` (опционально) |
| `address` | строка | Адрес (пусто для «Выезда») |
| `capacity` | объект | `min`, `max` — количество человек |
| `directions` | массив строк | Доступные направления |
| `photo` | путь к файлу | Основное фото или галерея |
| `description` | текст | Полное описание места |
| `masters` | массив slug | Мастера, работающие на локации |

### Открытые вопросы по модели данных

- Формат хранения: MDX + frontmatter удобен для редакторов, JSON/TS — для типобезопасности в коде. Решить при настройке проекта.
- Общий пул услуг каждого направления: где хранится и в каком формате — отдельный файл данных направления или захардкожено в коде страницы. Решить до вёрстки `/banya`.
