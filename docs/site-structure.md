# Структура сайта «Бодрость и Сила»

Информационная архитектура и контент-план фазы 1. Без вёрстки и кода — только структура страниц, описание блоков и связи между ними.

---

## Навигация (основное меню)

```
Главная · Баня · Релаксология · Кунгфу · Мастера · Локации
```

Мастера доступны через меню и через карточки внутри разделов. QR-коды на визитках ведут напрямую на `/masters/[slug]` — страница мастера работает как самостоятельная точка входа.

---

## Карта сайта

```
/                     Главная (лендинг)
/banya                Баня
/relax                Релаксология
/kungfu               Кунгфу
/masters              Все мастера
/masters/[slug]       Страница мастера
/locations            Локации
/blog                 Статьи (фаза 1.5, опционально)
/blog/[slug]          Статья
```

## Страницы

### / — Главная

Лендинг. Знакомит с проектом и направляет дальше. Формы записи нет — только атмосфера и ориентиры.

#### Блок 1: Hero
- Полноэкранный или крупный визуальный блок.
- Фото или видео, передающее атмосферу: тело, пар, движение.
- Название проекта: **Бодрость и Сила**.
- Одна короткая фраза — не слоган, а суть. Что здесь есть.
- Без кнопки записи — пространство для погружения, не продажа.

#### Блок 2: Три направления
- Три карточки или секции: **Баня**, **Релаксология**, **Кунгфу**.
- Каждая — краткое описание (1–2 предложения) + ссылка на раздел.
- Без перечисления услуг, без цен. Только ощущение.
- Можно сочетать — между блоками деликатная перекрёстная связь: «хорошо работает вместе».

#### Блок 3: О нас
- Кто мы: команда, сообщество, подход.
- Что даём — без маркетинговых клише, живым языком.
- Мастера упоминаются как люди, не как «специалисты».
- Не объясняем зачем нужна баня или кунгфу — говорим о себе.

#### Блок 4: Соцсети
- Ссылки на активные каналы/группы.
- Короткий призыв: подписывайся, следи за нами.
- Можно вставить последний пост или фото как превью.

#### Блок 5: Поддержка
- Тон: мы в начале пути, нам важен каждый.
- Призыв: расскажи друзьям, поделись, запишись сам — это и есть поддержка.
- Без пафоса и давления. Короткий, человечный текст.
- Можно добавить: «Каждый подписчик, каждая запись — для нас это важно».

#### Блок 6: Футер
- Логотип, название.
- Навигация (дублирует меню).
- Ссылки на соцсети.
- Placeholder контактов (заменим перед публичным запуском).
- Юридический плейсхолдер — ИНН, адрес, форма собственности (реальные данные позже).

### /banya — Баня

Страница направления. Рассказывает что такое банное направление, что входит, сколько стоит, кто работает.

#### Блок 1: Описание направления
- Заголовок: **Баня**.
- 2–4 предложения о том, что это и как это происходит. Без объяснений зачем.
- Форматы: выезд к клиенту, партнёрские бани, собственная локация.
- Вместимость: 1–15 человек, возможна работа в 4 руки.

#### Блок 2: Базовое парение
- Название: **Базовое парение**.
- Описание: приход, работа с паром, веник.
- Это основа — на что всё остальное нанизывается.

#### Блок 3: Допники
- Список дополнительных опций с кратким описанием каждой:
  - Выбор веника (берёза, дуб, эвкалипт, хвоя и др.)
  - Скрабирование
  - Ароматерапия
  - Звукотерапия (поющие чаши)
  - Чай / травяной напиток
  - Работа с телом после пара (тактильное восстановление)
- Не прайс — описание ощущений и смысла каждого.

#### Блок 4: Программы
- Готовые комбинации для тех, кто не хочет собирать вручную:

| Программа | Состав |
|---|---|
| Классика | Парение + веник на выбор + чай |
| Восстановление | Парение + скрабирование + работа с телом после пара + чай |
| Сенсорная | Парение + ароматерапия + звукотерапия |
| В 4 руки | Парение + работа с телом двумя мастерами |
| Полный релакс | Все допники включены |
| Компания | До 15 человек, базовое + опции под группу |

#### Блок 5: Конфигуратор стоимости
- Цена зависит от: количества человек + набор допников.
- Интерактивный элемент: выбираешь количество людей и допники — видишь итоговую цену.
- Не жёсткий прайс-лист — гибкий расчёт.
- После конфигурации — кнопка «Оставить заявку» с передачей выбранного состава в форму.

#### Блок 6: Мастера этого направления
- Лента карточек пар-мастеров.
- Карточка: фото, имя, короткое описание, ссылка на `/masters/[slug]`.
- Не вложенная структура — мастера показываются горизонтально рядом с услугами.

#### Блок 7: Форма заявки
- Заголовок: «Записаться».
- Поля: имя, телефон, пожелания (опционально), выбранная программа (если пришёл из конфигуратора — подставляется автоматически).
- Заявка уходит в Telegram-бот, мы перезваниваем.
- Никаких внешних систем записи на этом этапе.

#### Перекрёстные ссылки
- «Хорошо сочетается с релаксологией» → /relax
- Карточки мастеров → /masters/[slug]
- Программы с упоминанием локаций → /locations

### /relax — Релаксология

Страница направления. Важно: слово «массаж» не используется нигде.

#### Блок 1: Описание направления
- Заголовок: **Релаксология**.
- Что это: тактильные практики, работа с телом, нейрорелаксация.
- Без медицинских формулировок, без объяснений зачем.
- Форматы: выезд, приезд к нам, допродажа к банному сеансу.

#### Блок 2: Базовая услуга
- Название: **Нейрорелаксация** / **Работа с телом** (уточнить с мастерами).
- Описание ощущений и подхода.
- Длительность и базовый формат.

#### Блок 3: Допники
- Список уточняется с мастерами перед запуском.
- Структура аналогична банному разделу.

#### Блок 4: Мастера этого направления
- Лента карточек релаксологов.
- Карточка: фото, имя, короткое описание, ссылка на профиль.

#### Блок 5: Форма заявки
- Аналогична банной. Поля: имя, телефон, пожелания.

#### Перекрёстные ссылки
- «Хорошо сочетается с баней» → /banya
- Карточки мастеров → /masters/[slug]

### /kungfu — Кунгфу

Страница направления. Запись пока недоступна — форма ожидания.

#### Блок 1: Описание направления
- Заголовок: **Кунгфу**.
- Что это, почему это особенное — есть то, чего нет у других в регионе.

#### Блок 5: Расписание
- Заглушка: «Расписание в разработке».
- Без конкретных дат — не обещаем то, чего нет.

#### Блок 6: Форма ожидания

- Поля: имя, телефон / email.

### /masters/[slug] — Страница мастера

Самостоятельная точка входа. Работает через QR-код на визитке без необходимости знать остальной сайт.

#### Блок 1: Профиль
- Имя мастера (крупно).
- Фото.
- Теги направлений: Баня / Релаксология / Кунгфу.
- Описание: кто такой, подход, характер работы. Живым языком, от первого лица.

#### Блок 2: Услуги
- Заголовок: **Чем занимаюсь**.
- Список услуг этого мастера (подмножество из общего пула направления).
- Без прайса — описания.

#### Блок 3: Форматы работы
- Выезд к клиенту / партнёрская локация / собственная локация.
- Ссылки на конкретные локации если применимо → /locations.

#### Блок 4: Форма заявки
- Заголовок: «Записаться к [имя]».
- Поля: имя, телефон, пожелания.
- Заявка идёт в Telegram-бот с пометкой мастера.

#### Перекрёстные ссылки
- Теги направлений кликабельны → /banya, /relax, /kungfu.
- Локации кликабельны → /locations.

### /locations — Локации

Хаб. Рассказывает где физически происходит работа. Список меняется и дополняется — структура должна держать расширение.

#### Блок 1: Заголовок и навигация
- Заголовок: **Локации**.
- Внутренняя навигация/фильтр: **Свои · Партнёрские · Выезд**.

#### Блок 2: Карточки локаций

**Самолётная 59** (своя, в разработке)
- Тип: своя локация.
- Статус: в ремонте, скоро открываем.
- Направления: баня (малые группы до 4 чел), тренировки, релаксология.
- Описание: когда будет готово, как будет выглядеть.

**Лукоморье** (партнёрская)
- Адрес: уточнить.
- Тип: партнёрский банный комплекс, усадьба за городом.
- Вместимость: до 15 человек.
- Уровень: премиум — хорошая отделка, VIP условия.
- Направления: баня.

**Китуся** (партнёрская)
- Тип: партнёрские бани, в черте города.
- Вместимость: до 8 человек.
- Уровень: бюджетный — просто попариться.
- Направления: баня.

**Оазис отдыха** (партнёрская)
- Адрес: ул. Полевая 23/1.
- Тип: за городом, домики с купелями.
- Вместимость: до 20 человек.
- Уровень: атмосферное место, описание уточняется.
- Направления: баня.

**Выезд к клиенту**
- Не привязан к адресу.
- Направления: релаксология (основной формат), баня (при наличии бани у клиента).
- Описание: мастер приезжает, всё необходимое с собой.

#### Блок 3: «Нет подходящего?»
- Короткий текст: «Работаем на выезд, обсудим твой вариант».
- Форма заявки или ссылка на форму.

#### Перекрёстные ссылки
- Каждая локация → список мастеров, работающих там → /masters/[slug].
- Каждая локация → доступные направления → /banya, /relax, /kungfu.

### /blog — Статьи (фаза 1.5)

Необязателен на старте. Структура заложена, включается без переделки остального сайта.

- Список статей/событий в обратном хронологическом порядке.
- Карточка: заголовок, дата, короткое превью, тег (баня / релаксология / кунгфу / событие).
- Статьи в MDX — редактируются как текстовые файлы, без CMS.
- Контент пишется как черновик, проходит ревью, только потом публикуется.

## Форма заявки

Единый механизм CTA. Присутствует на страницах: направлений (Баня, Релаксология), мастеров, локаций.

- **Поля:** Имя, Телефон, Пожелания (опционально).
- **Контекст:** передаётся скрытым полем — откуда пришла заявка (направление, мастер, локация, программа из конфигуратора).
- **Обработка:** заявка уходит в Telegram-бот, менеджер перезванивает и согласовывает время вручную.
- **Никаких внешних систем записи** на этапе MVP.

Кунгфу — отдельная форма ожидания: имя + телефон/email, уведомление когда откроется запись.

## Что не входит в фазу 1

- Онлайн-запись через YClients или аналоги
- Личный кабинет клиента
- Магазин
- Партнёрские страницы
- Расписание тренировок
- Чат-бот на главной (планируется отдельным этапом)
- Реальные юридические данные (заменим перед публичным запуском)
