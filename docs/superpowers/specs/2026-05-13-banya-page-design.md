# Страница `/banya` — дизайн

**Date:** 2026-05-13
**Goal:** Сделать первую страницу направления — `/banya`. Это шаблон, по которому потом сделаем `/relax` и `/training`.

## Контекст и источники

- **Содержание блоков и порядок** — `docs/site-structure.md` §«/banya — Баня» (87–149).
- **Тон, форматы, допники, программы** — `docs/concept.md` §«Направление: Баня».
- **Конфигуратор** — без цен (см. Q2 в брейнсторме). Собирает состав, подставляет в форму. Цена «согласуется индивидуально, мы перезвоним».
- **Скоуп** — только `/banya`. `/relax`, `/training` — отдельными итерациями (см. Q1).

## Структура страницы

Сверху вниз. Каждая секция — отдельный компонент, мосты-градиенты как на главной.

1. **`BanyaHero`** — под-страничный hero. Breadcrumb «Главная / Баня», крупный H1 «Баня», lead-абзац. Тёмная neutral-зона.
2. **`BanyaIntro`** — описание направления (2–4 предложения), форматы (выезд / партнёрские / своя), вместимость (1–15 человек, 4 руки). Тёплая зона.
3. **`BanyaPricingNote`** — короткая фраза-успокоение: «Цена согласуется индивидуально — соберите состав в конфигураторе, и мы назовём ориентир». Тёплая зона, маленький блок.
4. **`BanyaBaseSteam`** — «Базовое парение» как один блок-манифест. Что это, на что нанизывается остальное. Тёплая зона.
5. **`BanyaAddons`** — 6 допников в cards-grid: веник (с описанием выборов), скрабирование, ароматерапия, звукотерапия, чай, работа с телом после пара. Тёплая зона.
6. **`BanyaPrograms`** — 6 готовых программ в format-list (Классика, Восстановление, Сенсорная, В 4 руки, Полный релакс, Компания). Каждая строка — название + состав. Тёплая зона.
7. **`BanyaConfigurator`** — интерактив. Тёплая зона, продолжается из предыдущих блоков. Внутри блок-карточка с границей и слегка более светлым/контрастным фоном — чтобы выделить из потока, но без смены зоны (мосты вокруг не нужны).
8. **`BanyaMasters`** — лента мастеров с тегом «баня». Тёплая зона.
9. **`BanyaCrossLinks`** — два крупных кросс-линка: «Хорошо сочетается с релаксологией» → `/relax`, «Где это происходит» → `/locations`. Прохладная зона.
10. **`BanyaContact`** — форма заявки. Принимает дефолтные значения из конфигуратора (направление=«Баня», message=сводка состава). Нейтральная зона.
11. **`Footer`** — реюз с главной. Тёмная зона.

Мосты-градиенты используем те же, что на главной: `bridge-warm-in`, `bridge-warm-to-cool`, `bridge-cool-to-darkneutral` и т.д. — комбинируя под последовательность зон страницы.

## Конфигуратор

**Что собирает:**
- Количество человек: 1–15 (счётчик +/−, default 1).
- Допники: 6 чекбоксов.
- Программа (опционально): один из 6 вариантов. При выборе программы — нужные допники подсвечиваются как «входят в программу», но остаются редактируемыми (пользователь может добавить ещё или убрать).

**Что выдаёт:**
- Сводка-«рецепт» одной строкой через `·`: например, «Парение · 4 человека · веник: дуб · скрабирование · работа с телом после пара · чай».
- Кнопка «Оставить заявку» — скроллит к форме и предзаполняет поле «Пожелания» этой сводкой; в поле «Направление» — «Баня».

**Что НЕ делает:**
- Не показывает цены. Никакие.
- Не сохраняет выбор между сессиями (никакого localStorage).
- Не валидирует количество людей против чего-то — просто число.

**Реализация:**
- React `useState` для people, addons, programChoice.
- Конфигуратор и форма — соседи на странице, общий клиентский parent с подъёмом state, либо через простой client-context. Конкретику решаем в плане.
- Передача в форму: при клике «Оставить заявку» — `scrollIntoView` на якорь формы + локальная установка `defaultMessage` через state, либо передача через query-параметр URL. В плане выберем минимально-инвазивный путь.

## Данные мастеров

Сейчас массив `MASTERS` хардкоднут в `app/_components/Masters.tsx`. Для `/banya` нужен тот же набор, отфильтрованный по «баня».

**Решение:** вынести в `app/_lib/data/masters.ts` как единый источник. Поменять модель:

```ts
export type Master = {
  name: string;
  role: string;
  directions: ("banya" | "relax" | "training")[];  // структурно
  tagsDisplay: string;                              // для UI как сейчас
  bio: string;
};
```

`Masters.tsx` начинает принимать пропсы:
- `title?: string` (default `"Мастера"`) — H2 заголовок секции
- `filterDirection?: "banya" | "relax" | "training" | null` (default `null` — все)
- `ctaHref?: string` (default `"/masters"`)
- `ctaLabel?: string` (default `"Все мастера →"`)

Главная страница импортит без пропсов — поведение не меняется. `/banya` — с `filterDirection="banya"`, title оставляем `"Мастера"` (тот же что на главной).

После фильтрации на /banya останется 3 мастера (Дмитрий, Александр, Вадим). Для marquee этого хватает (длина ленты × 2 = 6 карточек).

## CSS-примитивы под-страниц

В исходном прототипе Claude Design были стили для под-страниц, я их **не** портировал в `app/globals.css` (когда делали главную). Возвращаю в этой итерации:

- `.hero-sub` + `.hero-sub .breadcrumb` + `.hero-sub h1` + `.hero-sub .lead` — под-страничный hero.
- `.manifesto` + `.manifesto .lead-q` + `.manifesto .body` + `.manifesto .body p.kicker` — двухколоночный лонг-форм (используется в Intro, BaseSteam).
- `.cards` + `.cards.col-2` + `.cards.col-3` + `.card` (с card-num, h3, p, meta-line) — карточки допников.
- `.format-list` + `.format-row` + `.fr-num` + `.fr-title` + `.fr-desc` + `.fr-price` (но без `.fr-price` — у нас нет цен) — программы.
- `.crosslinks` + `.crosslink` + `.crosslink.to-cool` + `.crosslink.to-neutral` — кросс-линки.

Все эти примитивы — в `app/globals.css`, конкретные блоки (1180px max-width, шрифты Lora/Inter, цвета зон) совпадают с уже существующими.

## Out of scope

- `/relax`, `/training` — отдельной итерацией.
- `/masters`, `/masters/[slug]`, `/locations`, `/locations/[slug]` — отдельной итерацией (на /banya cross-links будут вести на пока-несуществующие маршруты, это ОК — добавим страницы позже).
- Цены любого вида.
- Серверная обработка формы. Сейчас `alert` как заглушка.
- Sticky-конфигуратор, drawer, модалки, sticky-CTA. Простой inline-блок в потоке страницы.
- `prefers-reduced-motion` оптимизации сверх того что уже есть в globals.css.
- SEO-метаданные сверх минимальных `<title>` и `<meta description>`.

## Тестирование

Тестов нет (фреймворк в проекте не настроен). Верификация:
- `npx tsc --noEmit` — типы.
- `npm run build` — должен пройти, `/banya` помечена `○ (Static)`.
- `curl http://localhost:3000/banya` — присутствует разметка всех блоков.
- Визуальный обход в браузере — десктоп и мобильный (узкий вьюпорт), все блоки читаются, конфигуратор работает, форма получает сводку.
