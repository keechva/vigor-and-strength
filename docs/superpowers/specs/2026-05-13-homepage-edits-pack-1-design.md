# Пакет правок главной №1 — дизайн

**Date:** 2026-05-13
**Goal:** Сделать четыре независимые правки главной страницы — поведение хедера, выравнивание hero-subtitle, чистка meta-слота секций направлений, фикс переноса слова «Тренировки».

## Контекст

Главная страница (`app/page.tsx`) собрана из секционных компонентов (`Hero`, `Banya`, `Relax`, `Training`, `Masters`, `Locations`, `Contact`, `Footer`) с глобальным `<Header />` в `app/layout.tsx`. Хедер — client-компонент, отслеживает hero через `IntersectionObserver`. Все четыре правки касаются только главной (`/`) и поведения хедера; других страниц пока нет.

## Scope

### 1. Хедер — slide-in после hero

**Текущее поведение:** хедер всегда виден; над hero — прозрачный фон, после hero — заливка `var(--neutral-bg)`.

**Новое поведение:** над hero — хедер полностью скрыт (`transform: translateY(-100%)`). Когда `.hero` перестаёт пересекаться с viewport (через тот же `IntersectionObserver`) — хедер въезжает сверху с тёмной заливкой; над hero его не видно вообще.

**Детали:**
- Единое состояние `is-revealed` управляет и трансформом, и фоном. Класс `is-filled` упраздняется — он больше не нужен, потому что «виден» = «залит».
- Transition: `transform 320ms cubic-bezier(0.25, 0.46, 0.45, 0.94)`. Прозрачный/полупрозрачный state над hero не нужен — хедер просто не присутствует на экране.
- На страницах без `.hero` (будущие /banya, /relax) хедер инициализируется в `is-revealed` (видимым).
- Sub-pages кейс: SSR/CSR флэш обычно при `useState(true)` ⇒ начальный transparent. Теперь начальный state = `false` ⇒ начальный hidden ⇒ на homepage срабатывает IntersectionObserver корректно (hero виден ⇒ остаётся скрытым). На sub-pages эффект сразу выставляет `true`.
- Мобильный бургер живёт внутри хедера — пока хедер скрыт, бургер тоже не виден. Это ОК: на hero пользователь не нуждается в навигации (контент сосредоточен), а ниже хедер появляется.
- A11y: при скрытом хедере его ссылки не должны быть в tab order. Делаем через `visibility: hidden` в скрытом state (по аналогии с мобильным оверлеем), с задержкой transition чтобы slide-out успел отыграть.

**Acceptance criteria:**
- На загрузке главной хедер не виден ни визуально, ни в tab order.
- При скролле hero уходит наверх → хедер плавно въезжает с тёмной заливкой за ~300мс.
- Скролл обратно вверх → хедер уезжает.
- На сабпейдже (если она появится; сейчас протестировать нечем) хедер виден сразу.

### 2. Hero — directions left-aligned

**Текущее:** строка `Баня · Релаксология · Тренировки` под H1 — `text-align: right` (унаследовано от хедер-варианта A).

**Новое:** `text-align: left`. H1 остаётся справа. Создаёт лёгкую визуальную тяжесть «имя справа — список слева».

**Детали:**
- Затрагивает только селектор `.hero .directions` (был `body[data-hero="A"] .hero .directions { text-align: right }` — теперь правило не нужно, либо явно `.hero .directions { text-align: left }`).
- Цвет dot-разделителей, размер шрифта, margin — не трогаем.

**Acceptance criteria:**
- На любом вьюпорте `Баня · Релаксология · Тренировки` ориентирована по левому краю обёртки `.wrap`.
- H1 остаётся справа.

### 3. Meta-слот секций направлений — только номер

**Текущее:** в `Banya.tsx`, `Relax.tsx`, `Training.tsx` метка `<div className="meta">01 · Направление</div>` (02/03 соответственно).

**Новое:** `<div className="meta">01</div>` (02/03). Слово «Направление» уходит, разделитель `·` тоже не нужен (нет второго токена).

**Детали:**
- Затрагивает только три компонента секций направлений.
- В `Masters.tsx` («Команда»), `Locations.tsx` («Где это происходит»), `Contact.tsx` («Связаться») — meta не трогаем.
- CSS селектор `.zone .meta` остаётся.

**Acceptance criteria:**
- Над H2 трёх direction-секций виден только номер (01/02/03) — без слова и без точки-разделителя.
- Над H2 в Мастерах/Локациях/Контактах — текст как был.

### 4. «Тренировки» без переноса

**Текущее:** `<h2>Трени&shy;ровки</h2>` (soft hyphen) в `Training.tsx`. На узких вьюпортах слово рвётся.

**Новое:** `<h2>Тренировки</h2>` — без soft hyphen.

**Детали:**
- Чтобы гарантировать, что слово не сломается даже при очень узком вьюпорте: `.practice.training h2 { text-wrap: nowrap }` в globals.css.
- На очень узких экранах (~280px и уже) это может вытолкнуть h2 за границу. Это приемлемая жертва — таких ширин у целевой аудитории нет.

**Acceptance criteria:**
- На любом вьюпорте слово «Тренировки» отрисовано одной строкой без дефиса.
- На стандартных мобильных (≥360px) ширины h2 хватает.

## Out of scope

- Не трогаем `.topnav` устаревшие стили (стоят отдельно по плану §11 предыдущего sticky-header плана).
- Не трогаем SSR-флэш на гипотетических sub-pages — отдельный вопрос когда они появятся.
- Не делаем перенос hero-варианты B/C/D обратно — оставляем только A.
- Не меняем поведение мобильного бургер-меню (открывается так же, закрывается так же).

## Архитектурные заметки

**Хедер state-machine** становится бинарным вместо тернарного:
- Был: `over-hero` (transparent, visible), `past-hero` (filled, visible).
- Стал: `over-hero` (hidden), `past-hero` (revealed = filled + slide-in done).

Это упрощает CSS — один класс `is-revealed` правит и transform, и background. Логика IntersectionObserver не меняется.

**Один коммит или раздельные:** разделим на 2 коммита — (a) Хедер (изменение поведения, новое имя класса, фиксы CSS под скрытие); (b) Hero/Sections/Training (мелкие правки разметки и CSS). Это даёт чистую историю и облегчает revert если slide-in поведение не зайдёт.
