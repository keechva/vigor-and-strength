# Концепция проекта "Бодрость и Сила"

## Суть проекта

Сайт-каталог мастеров и услуг. Три направления: **баня**, **релаксология**, **кунгфу**. Название говорит само за себя — объяснять ничего не надо. Человек приходит за бодростью и силой — вот что здесь есть.

Направления можно сочетать. Баня хорошо работает с релаксологией. Тренировки дополняют восстановление. Это комплексный подход, но без навязывания — каждый берёт своё.

## Тон и подача

- Спокойный, прямой, без пустых слов
- Никакого маркетингового мусора и глянца
- Живые мастера, живые ощущения
- Не объясняем зачем это нужно — показываем что есть

## Терминология

Слово "массаж" не используем — это медицинский термин, медицинской лицензии нет.

Используем:
- **Релаксология** — название направления
- **Нейрорелаксация**, **тактильные практики**, **работа с телом** — описание услуг
- **Релаксолог** — мастер этого направления
- **Тактильное восстановление**, **работа с телом после пара** — в банном контексте

## Структура сайта

Сайт строится так, чтобы новые разделы добавлялись без переделки существующих.

### Фаза 1 (текущая)
```
Главная
├── Баня
│   ├── Базовое парение
│   ├── Допники
│   ├── Программы
│   └── Мастера (пар-мастера)
├── Релаксология
│   ├── Базовая услуга
│   ├── Допники
│   └── Мастера (релаксологи)
├── Кунгфу
│   ├── Индивидуальные тренировки
│   ├── Групповые занятия
│   └── Расписание (в разработке)
└── Локации (список партнёрских и своих)
```

### Заложено на будущее (фаза 2+)
```
├── Магазин (банные аксессуары и сопутствующее)
├── Партнёры (партнёрские странички)
└── События (ретриты, мероприятия)
```

Переходы: между направлениями — перекрёстные ссылки ("хорошо сочетается с..."). Где уместно — готовые комбо-программы.

## Воронка

1. Человек заходит на главную → видит три направления
2. Выбирает направление → видит базовую услугу + допники + мастеров
3. Может: сконфигурировать программу / выбрать мастера / посмотреть комбо
4. CTA: записаться (YClients) или перейти в соцсети

## Направление: Баня

### Формат работы
- Выезд к клиенту
- Партнёрские бани (запись к пар-мастеру)
- Собственные бани (в разработке)
- 1–15 человек, возможна работа в 4 руки (два мастера)

### Базовое парение
Приход, парение с веником.

### Допники
- Выбор веника (берёза, дуб, эвкалипт, хвоя и др.)
- Скрабирование
- Ароматерапия
- Звукотерапия (поющие чаши)
- Чай / травяной напиток
- Работа с телом после пара (тактильное восстановление)

### Черновые программы

| Программа | Состав |
|---|---|
| Классика | Парение + веник на выбор + чай |
| Восстановление | Парение + скрабирование + работа с телом после пара + чай |
| Сенсорная | Парение + ароматерапия + звукотерапия (поющие чаши) |
| В 4 руки | Парение + работа с телом двумя мастерами |
| Полный релакс | Все допники включены |
| Компания | До 15 человек, базовое + опции под группу |

### Ценообразование
Цена зависит от количества человек и набора допников. На сайте нужен **конфигуратор / калькулятор**.

### Допники
*(уточнить у мастеров)*

Релаксологи. У каждого — своя страница, свой набор услуг.

## Направление: Кунгфу

Основное направление тренировок. Есть то, чего нет у других в регионе.

- Индивидуальные тренировки
- Групповые занятия
- Расписание — в разработке, не на первом этапе

## Модель данных

Сущности независимы и связываются через теги/ссылки — не через вложенность.

### Мастер
- Имя, фото, описание
- Теги направлений (баня / релаксология / кунгфу / ...) — может быть несколько
- Свой набор услуг (подмножество из общего пула)
- Форматы работы (выезд / партнёрская баня / своя локация)
- CTA: записаться

### Направление
- Название, описание
- Пул базовых услуг
- Пул допников
- Программы / комбо

### Услуга
- Название, описание
- Принадлежность к направлению(ям)
- Тип: базовая / допник

### Локация
- Название, адрес, тип (своя / партнёрская)
- Мастера, работающие на ней

### Связи
```
Мастер ←→ Направления  (теги, многие-ко-многим)
Мастер ←→ Услуги       (личный набор, подмножество пула)
Мастер ←→ Локации      (где работает)
Услуга ←→ Направление  (принадлежность)
```

Такая модель позволяет показывать мастеров в разных разрезах: по направлению, по локации, по услуге — без переделки структуры.

Сейчас — плейсхолдеры. Реальные данные добавим перед публичным запуском.

## Гибкость архитектуры

- Направления добавляются/убираются независимо
- Мастера добавляются/убираются независимо
- Допники и программы настраиваются на уровне контента
- Новые разделы (магазин, партнёры, события) подключаются как отдельные модули без переделки существующего
- Локации — отдельный справочник, не привязанный жёстко к направлениям

## Что не делаем

- Не используем слово "массаж"
- Не пишем пустые маркетинговые слоганы
- Не объясняем зачем нужна баня или тренировки
- Не публикуем реальные юридические данные до запуска
