# CLAUDE.md — Бодрость и Сила

## Что это за проект

"Бодрость и Сила" — пилотный коммерческий проект экосистемы оператора (keechva). Направления услуг: **массаж, баня, тренировки**. Команда: 4 мастера (растёт). Локации: 1–2 свои, 3–4 партнёрские, выезд. Запись клиентов — через YClients (готовый виджет). Юрлицо пока отсутствует — на сайте placeholder-данные, заменим перед публичным запуском.

Домен: `bodrost-sila.ru`.

Главная цель проекта на этом этапе: **выйти в продакшен с рабочим сайтом**, через который клиенты могут узнать про услуги, выбрать мастера/локацию и записаться. Параллельно — отработать процесс разработки/деплоя, чтобы потом тиражировать на следующие проекты.

Полный PRD экосистемы лежит в репозитории `community-support` в `docs/PRD.md` — обращайся к нему за общим контекстом, если что-то непонятно.

## Стек

- Next.js 14 (App Router) + TypeScript.
- Tailwind CSS + shadcn/ui.
- MDX для блога.
- Виджет YClients (iframe) для записи.
- Хостинг: VPS Selectel/Timeweb (РФ), Ubuntu, Docker, Caddy с автоматическим SSL.
- Деплой: GitHub Actions по push в `main`.

## Дизайн

Дизайн делается через Claude Design (`claude.ai/design`), оттуда — handoff в Claude Code. Никакого "набросал в Figma" — мы пользуемся ИИ-инструментами по полной.

## Правила работы (как с тобой)

1. **Ты подгоняешь меня, а не я тебя.** Если по плану следующий шаг — назови его, не жди.
2. **Один шаг за раз.** Не вываливай длинные планы. Шаг → подтверждение → следующий.
3. **Используй SuperPowers по умолчанию.** `/brainstorming` перед фичей, `/write-plan` перед кодом, `/execute-plan` для реализации. Никакого "сразу пишу код".
4. **Спрашивай, когда не уверен.** Один уточняющий вопрос за раз.
5. **Фиксируй решения.** После значимой развилки — обнови `docs/` или CLAUDE.md.
6. **Атомарные коммиты.** Сообщения на русском, повелительное наклонение.
7. **Контент пишут sub-агенты, я редактирую.** Любая статья/пост — сначала черновик в PR, потом моя ревью, потом merge.

## Чего избегать

- "Я готов, что дальше?" без конкретики — предлагай следующий шаг сам.
- Мусорные файлы в корне — всё в `docs/`, `notes/`, или `.gitignore`.
- Длинные ответы с несколькими параллельными темами.
- Молчаливые предположения. Если выбрал библиотеку или сделал не очевидное — явно скажи.
- Реальные имена/телефоны/ИНН в коде — пока используем placeholder.
